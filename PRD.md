# PRD — Система лояльности Техноточка

## Описание продукта

Telegram Mini App — цифровая программа лояльности для комиссионного магазина техники «Техноточка».
Клиенты получают кешбэк баллами за покупки и тратят их при следующих визитах.
Сотрудники начисляют баллы прямо в Telegram без кассовых систем.
Администратор видит аналитику и управляет командой.

**Бот:** @technobonusbot  
**Админ Telegram ID:** 339860192  
**Хостинг:** https://browniriz.github.io/technotochka-loyalty/

---

## Пользователи и роли

### Клиент
Любой человек, который впервые написал боту или перешёл по реферальной ссылке сотрудника.
Идентифицируется по Telegram ID. При первом входе проходит регистрацию (ФИО + телефон).

### Сотрудник
Сотрудник магазина. Добавляется администратором вручную (по Telegram ID).
Видит панель сотрудника вместо кабинета клиента.

### Администратор
Владелец или руководитель. Telegram ID: 339860192.
Имеет доступ ко всей системе и панели управления.

## Уровни лояльности

| Уровень | Сумма покупок за год | Кешбэк |
|---|---|---|
| Новый | 0 — 10 000 ₽ | 3% |
| Постоянный | 10 000 — 40 000 ₽ | 5% |
| Свой | от 40 000 ₽ | 7% |

- 1 балл = 1 рубль
- Списание: до 15% от суммы покупки
- Уровень пересчитывается автоматически после каждой продажи
- Накопленная сумма считается за календарный год

## Экраны и сценарии

### Экран 0 — Загрузка и определение роли

**Сценарий:**
1. Пользователь открывает Mini App
2. Показывается экран загрузки (лого + спиннер)
3. Запрос к API `getRole` с Telegram ID
4. В зависимости от роли — переход на нужный экран

**Правила:**
- Если роль `admin` → показать панель администратора
- Если роль `staff` → показать панель сотрудника
- Если роль `client` → проверить регистрацию (`getClient`)
  - Если клиент найден → показать кабинет клиента
  - Если не найден → показать экран регистрации

---

### Экран 1 — Регистрация клиента

**Показывается:** новому пользователю при первом входе

**Элементы:**
- Заголовок: «Добро пожаловать в Техноточку»
- Подзаголовок: «Зарегистрируйтесь чтобы получать бонусы»
- Поле: ФИО (обязательное)
- Поле: Номер телефона (обязательное, формат +7...)
- Кнопка: «Зарегистрироваться»

**Логика:**
- После отправки → вызов API `registerClient`
- Если пришёл по реферальной ссылке (`?start=staff_ID`) → передать `referred_by` в API
- После успешной регистрации → переход в кабинет клиента

---

### Экран 2 — Кабинет клиента

**Показывается:** зарегистрированному клиенту

**Элементы:**
- Шапка: имя клиента + уровень лояльности (цветной бейдж)
- Карточка баллов: большое число баллов на счёте + подпись «≈ X ₽»
- Прогресс до следующего уровня: прогресс-бар + текст «До уровня Постоянный — X ₽»
  - На уровне «Свой» показывать: «Максимальный уровень достигнут»
- Две мини-карточки рядом: «Куплено за год» и «Следующий уровень»
- Раздел «История покупок»: список последних 10 покупок
  - Каждая строка: название-заглушка или дата + сумма + начислено баллов
- Кнопка внизу: «Показать QR-код»

**Логика QR:**
- По нажатию → полноэкранный оверлей с QR-кодом
- QR содержит Telegram ID клиента
- Генерируется через qrcode.min.js прямо в браузере
- Кнопка закрыть оверлей

---

### Экран 3 — Панель сотрудника

**Показывается:** сотруднику (role = staff)

**Секция 1 — Поиск клиента:**
- Поле поиска с placeholder «Имя, телефон или Telegram ID»
- Выпадающий список результатов при вводе от 2 символов
- Кнопка «Сканировать QR» → открывает `Telegram.WebApp.showScanQrPopup()`
- После сканирования QR → автоматически находит клиента

**Секция 2 — Найденный клиент:**
- Карточка: ФИО + телефон + баланс баллов + уровень
- Показывается после выбора из поиска или сканирования QR

**Секция 3 — Оформление продажи:**
- Поле: «Сумма покупки, ₽»
- Поле: «Списать баллы» (необязательное, макс. 15% от суммы)
- Превью начисления: «Будет начислено: X баллов (Y%)»
- Кнопка: «Начислить баллы»
- После успеха → уведомление «Начислено X б клиенту [ФИО]» + сброс формы

**Секция 4 — Реферальная ссылка:**
- Текст: «Ваша реферальная ссылка»
- Ссылка: `https://t.me/technobonusbot?start=staff_[tg_id]`
- Кнопка «Скопировать»

---

### Экран 4 — Панель администратора

**Показывается:** администратору (role = admin)

**Секция 1 — Статистика (3 карточки):**
- Всего клиентов
- Продаж за месяц (количество + сумма в ₽)
- Баллов выдано всего

**Секция 2 — Распределение по уровням:**
- Три числа: Новый / Постоянный / Свой

**Секция 3 — Рейтинг сотрудников:**
- Список сотрудников, отсортированный по сумме продаж за месяц
- Каждая строка: имя + город + продажи за месяц (кол-во и сумма) + приглашено клиентов
- Бейдж «admin» у администраторов

**Секция 4 — Управление сотрудниками:**
- Кнопка «Добавить сотрудника»
- Форма: Telegram ID + Имя + Город + Роль (staff / admin)
- Кнопка удаления у каждого сотрудника (кроме себя)

## Навигация

- Клиент: один экран, кнопка QR открывает оверлей
- Сотрудник: один экран с секциями, скролл вниз
- Администратор: один экран с секциями, скролл вниз
- Нет вкладок и меню — всё на одной странице

## API — действия

| action | Кто вызывает | Что делает |
|---|---|---|
| getRole | все | определить роль по tg_id |
| getClient | клиент | получить профиль + историю |
| registerClient | клиент | зарегистрировать нового клиента |
| searchClient | сотрудник | поиск по ФИО / телефону / ID |
| getClientById | сотрудник | найти клиента после сканирования QR |
| addSale | сотрудник | оформить продажу, начислить баллы |
| getAdminStats | админ | общая статистика |
| getStaffList | админ | список сотрудников с показателями |
| addStaff | админ | добавить сотрудника |
| removeStaff | админ | удалить сотрудника |

## Что НЕ входит в MVP

- SMS-уведомления клиентам
- Бронирование товара
- Ранний доступ к поступлениям
- Скидки на диагностику
- Push-уведомления
- Личный менеджер
- Мультиязычность
- Веб-версия вне Telegram
